# Phase 1: Foundation - Dockerization Plan

I will now execute the Dockerization of the `HyperTutorialProject` CLI.

## 1. Create `.dockerignore`
*   **Purpose:** Exclude heavy/sensitive folders (`node_modules`, `.env`, `.git`) from the build context.
*   **Location:** Project root.

## 2. Create `Dockerfile`
*   **Strategy:** Single-stage build (for simplicity in dev mode) using `node:18-alpine`.
*   **Key Detail:** We will install *all* dependencies (including `devDependencies`) because the current CLI execution relies on `tsx` (which is a dev dependency).
*   **Command:** `npm run power-moves`

## 3. Create `docker-compose.yml`
*   **Purpose:** Orchestrate the run configuration.
*   **Features:**
    *   **Interactive Mode:** `stdin_open: true` and `tty: true` to support the Inquirer prompts.
    *   **Env Injection:** Loads from `.env`.
    *   **Volume Mounts:** Mounts local config for seamless updates.

## 4. Documentation
*   Add a section to `README.md` explaining how to run with Docker:
    ```bash
    docker-compose run --rm hyper-cli
    ```

**Verification:**
After creating the files, I will verify the setup by running a `docker build` check (dry run command proposal) or explaining the verification steps.
